Allatori Maven Plugin

4158字 2026-03-06
18 min read
Scroll Down

Allatori Maven Plugin 是一个用于 Allatori Java 混淆器 的 Maven 插件，在 Maven 构建过程中自动执行 Java 字节码混淆，无需单独安装 Allatori 或手写命令行。

一、设计原理

1.1 为什么需要本插件

  • Allatori 官方用法：通常需要单独下载 Allatori、配置 allatori.xml，再在命令行执行 java -jar allatori.jar config.xml，且配置中的路径往往需要与 Maven 的 target、finalName 等保持一致。
  • 本插件的目标：把“读配置 → 替换 Maven 变量 → 调用 Allatori”这一流程固化到 Maven 的 package 阶段，实现“打包即混淆”，并支持 ${var} 与 @var@ 两种变量格式，便于与标准 Maven 及 Spring Boot 资源过滤习惯一致。

1.2 与 Maven 生命周期的关系

  • 插件唯一 Goal：obfuscate，绑定在 LifecyclePhase.PACKAGE。
  • 执行顺序：在 package 阶段，Maven 会先完成 jar:jar（或 war:war 等），生成未混淆的产物（如 target/xxx.jar），随后执行本插件的 obfuscate，读取用户提供的 allatori.xml，替换其中的 Maven 变量，再以 子进程 方式调用 java -jar allatori.jar <处理后的配置文件>，在指定输出目录生成混淆后的 jar。
  • 设计要点：不修改 Maven 内置的打包逻辑，只在打包之后增加一步“混淆”，输入为已有 jar + 配置文件，输出为混淆后的 jar，便于与 CI/CD 和现有 pom 配置兼容。

1.3 变量替换与“单一事实来源”

  • 配置文件中往往需要“构建输出目录”“最终 jar 名”等，这些在 Maven 中已有唯一事实来源（如 project.build.directory、project.build.finalName）。
  • 插件在 不修改用户原始配置文件 的前提下，将配置内容读入内存，做 纯字符串级 Maven 变量替换（${...} 与 @...@），再写出到构建目录下的临时文件（allatori-processed.xml），并用该临时文件驱动 Allatori；不解析 XML 结构，不依赖 Allatori 的 schema，从而保证“路径与名称”全部来自 Maven，避免手写绝对路径或重复维护多份配置。
  • 替换后的配置文件完整内容会逐行输出到 Maven 日志，便于在 CI 或本地排查变量是否按预期替换。

1.4 Allatori 的提供方式

  • 内置模式：插件 resources 中可携带 lib/allatori.jar，运行时若未配置 jarFile 或指向的文件不存在，则从 classpath 解压到系统临时目录（如 java.io.tmpdir/allatori-maven-plugin/allatori.jar）再调用，执行完毕后可选清理临时文件。
  • 外置模式：用户通过参数 jarFile 指定本机已有的 Allatori jar，插件直接使用，便于升级 Allatori 版本

二、底层设计

2.1 整体架构

┌─────────────────────────────────────────────────────────────────────────┐
│                        Maven Build (package phase)                        │
└─────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  AllatoriMojo.execute()                                                  │
│  ┌──────────────┐   ┌─────────────────────┐   ┌──────────────────────┐  │
│  │ 解析/校验参数  │ → │ 解析 Allatori JAR   │ → │ 校验 config 与 outDir │  │
│  └──────────────┘   └─────────────────────┘   └──────────────────────┘  │
│         │                     │                           │               │
│         ▼                     ▼                           ▼               │
│  ┌──────────────┐   ┌─────────────────────┐   ┌──────────────────────┐  │
│  │ processConfig │   │ replaceMavenVars   │   │ executeAllatori       │  │
│  │ File()        │   │ (${var} / @var@)   │   │ (ProcessBuilder)     │  │
│  └──────────────┘   └─────────────────────┘   └──────────────────────┘  │
│         │                     │                           │               │
│         └─────────────────────┴───────────────────────────┘               │
│                                      │                                     │
│                      输出：allatori-processed.xml → Allatori 子进程        │
└─────────────────────────────────────────────────────────────────────────┘
  • 单一 Mojo：所有逻辑集中在 AllatoriMojo，通过 @Parameter 注入 project、configFile、jarFile、outputDirectory、skip，无额外 Mojo 或复杂依赖树。
  • 三步流程：① 确定 Allatori jar（内置解压或用户指定）→ ② 处理配置文件（UTF-8 读写 + 变量替换 + 写临时文件）→ ③ 以 ProcessBuilder 在 outputDirectory 下执行 java -jar allatori.jar <processed.xml>，并消费标准输出/错误，根据退出码决定成功或抛 MojoExecutionException。
  • 异常与清理：任一步骤抛出的 Exception 在 execute() 的 catch 中统一包装为 MojoExecutionException；仅当本次使用了“插件解压的” Allatori jar 时，在 finally 中尝试删除该临时 jar（失败仅打 warn）。

2.2 模块职责

模块 职责
allatori-maven-plugin 提供唯一 Goal obfuscate，实现配置处理与 Allatori 调用；resources 内可含 lib/allatori.jar 供内置使用。
allatori-plugin-test 独立 Maven 项目，依赖并绑定插件，用于人工/集成验证“打包 + 混淆”全流程。

2.3 数据流与文件约定

  • 输入：用户项目中的 allatori.xml（路径由 configFile 指定）、Maven project 属性、可选 jarFile。
  • 中间文件：<outputDirectory>/allatori-processed.xml，即替换变量后的配置；每次执行都会覆盖该文件，并在日志中逐行输出其完整内容（getLog().info(line)），便于排错。
  • 输出：由 Allatori 根据 allatori-processed.xml 中的 <input><jar out="..."/> 等配置写入，通常为同一 outputDirectory 下的混淆后 jar（如 *-obfuscated.jar）。
  • 工作目录：ProcessBuilder.directory(outputDirectory)，确保 Allatori 内相对路径与 Maven 构建目录一致。
  • 临时 Allatori jar（仅内置模式）：解压到 System.getProperty("java.io.tmpdir")/allatori-maven-plugin/allatori.jar；若该文件已存在则不会重复解压（多次构建可复用）；本次执行若为插件解压所得，则在 finally 中尝试删除并打日志（删除失败仅 warn，不抛错）。

2.4 扩展与约束

  • 扩展：如需支持更多变量，只需在 getMavenProperty(String) 中增加分支或从 project.getProperties() 读取；如需支持多配置文件或 profile，可在 Mojo 层增加参数与循环调用。
  • 约束：不在插件内实现混淆算法，不解析 Allatori 的 XML 语义，仅做字符串级变量替换与进程调用，保证“插件只做编排，混淆逻辑完全由 Allatori 负责”。

三、代码实现

3.1 核心类与注解

  • 类：com.dv.maven.plugin.AllatoriMojo，继承 org.apache.maven.plugin.AbstractMojo。
  • Goal 与阶段：@Mojo(name = "obfuscate", defaultPhase = LifecyclePhase.PACKAGE)，即执行 mvn package 时会触发 obfuscate。

3.2 参数（@Parameter）

参数 类型 说明 默认/必填 命令行覆盖属性
project MavenProject 当前 Maven 项目 ${project}，只读 —
configFile File Allatori 配置文件路径 必填，无默认 allatori.configFile
jarFile File Allatori 可执行 jar 路径 可选；不填或不存在则用内置 allatori.jarFile
outputDirectory File 混淆输出目录，兼作 Allatori 工作目录 ${project.build.directory} allatori.outputDirectory
skip boolean 是否跳过混淆 false allatori.skip
  • 命令行覆盖示例：mvn package -Dallatori.skip=true、mvn package -Dallatori.configFile=path/to/config.xml。

3.3 执行流程（execute）

  1. skip：若 skip == true，直接 return，不执行后续逻辑。
  2. 解析 Allatori jar：若 jarFile == null 或不存在，调用 extractAllatoriJar() 从 getClass().getResourceAsStream("/lib/allatori.jar") 解压到 java.io.tmpdir/allatori-maven-plugin/allatori.jar，并标记为临时文件以便 finally 中删除。
  3. 校验：检查 jarFile、configFile 存在性，若不存在则抛出 MojoExecutionException；确保 outputDirectory 存在（不存在则 mkdirs()）。
  4. 处理配置：processConfigFile() → 见下文。
  5. 执行 Allatori：executeAllatori(processedConfigFile) → 见下文。
  6. 清理：在 finally 中若存在“本次解压的”临时 jar，则尝试删除并打日志。

3.4 配置处理（processConfigFile）

  • 读取：Files.readAllBytes(configFile.toPath())，再以 UTF-8 解码为字符串（源码使用 "UTF-8" 字面量）。
  • 替换：调用 replaceMavenVariables(content) 得到新字符串（不修改原文件）。
  • 写出：将结果写入 <outputDirectory>/allatori-processed.xml，编码 UTF-8（content.getBytes("UTF-8")）；若文件已存在则覆盖。
  • 日志：写入后使用 BufferedReader + InputStreamReader(..., "UTF-8") 按行读取该文件，每一行通过 getLog().info(line) 输出到控制台，便于调试与审计。
  • 异常：任何 IOException 包装为 MojoExecutionException("Failed to process Allatori config file", e) 抛出。

3.5 变量替换（replaceMavenVariables）

  • 执行顺序：先处理所有 ${var}，再处理所有 @var@；两种格式互不嵌套解析。
  • ${var} 规则：从当前内容中找 "${"，再找其后第一个 "}"，中间子串为变量名；若 getMavenProperty(var) 非 null 则整体替换为值，并从替换后位置继续找下一个 "${"；否则从该 "}" 之后继续找。若找不到配对 "}" 则结束本轮。
  • @var@ 规则：找 "@"，再找其后下一个 "@"，且要求 end > start + 1（即排除 @@ 空变量）；中间子串为变量名，替换逻辑同 ${}。未匹配的占位符原样保留，不抛错。
  • 注意：嵌套写法如 ${a${b}} 会按“第一个 ${ 到第一个 }”截取，得到变量名 a${b，通常非预期；建议避免嵌套。

3.6 Maven 属性解析（getMavenProperty）

  • 解析顺序：① project.getProperties().getProperty(propertyName)（pom 的 <properties> 及 Maven 内置属性）；② System.getProperty(propertyName)；③ 内置 switch 写死的六项：project.basedir（project.getBasedir().getAbsolutePath()）、project.build.directory、project.build.outputDirectory、project.build.finalName、project.artifactId、project.version。
  • 返回值：均无则返回 null；调用方在 replaceMavenVariables 中不会替换该占位符（保留原样）。

3.7 执行 Allatori（executeAllatori）

  • 命令：java -Xms128m -Xmx512m -jar <jarFile 绝对路径> <processedConfigFile 绝对路径>；JVM 堆固定为 128m/512m，如需调整需改源码。
  • 进程：ProcessBuilder(command)，processBuilder.directory(outputDirectory)，redirectErrorStream(true)（标准错误合并到标准输出）。
  • 输出消费：启动后使用 BufferedReader(new InputStreamReader(process.getInputStream())) 逐行读取（未指定 charset，使用 JVM 默认编码），每行 getLog().info(line)，并拼入 StringBuilder 供失败时输出。
  • 退出码：process.waitFor()；若 exitCode != 0，先 getLog().error(...) 输出完整内容，再抛 MojoExecutionException("Allatori execution failed with exit code: " + exitCode + ". Output: " + output)。
  • 异常：IOException 或 InterruptedException 时包装为 MojoExecutionException("Failed to execute Allatori obfuscator", e)；若为 InterruptedException 会调用 Thread.currentThread().interrupt() 恢复中断状态。
  • 清理：finally 中若 process != null 则调用 process.destroy()（未使用 destroyForcibly()，为正常结束子进程）。

3.8 内置 jar 解压（extractAllatoriJar）

  • 临时目录：new File(System.getProperty("java.io.tmpdir"), "allatori-maven-plugin")，若不存在则 mkdirs()，失败抛 MojoExecutionException。
  • 目标文件：<tempDir>/allatori.jar。若该文件已存在则直接返回，不重复解压（多模块或多次构建可复用）。
  • 解压来源：getClass().getResourceAsStream("/lib/allatori.jar")；若为 null 抛 MojoExecutionException("Embedded Allatori jar not found in plugin resources")。
  • 拷贝：4KB 缓冲区，InputStream.read(buffer) → OutputStream.write(buffer, 0, bytesRead)，直到读尽。任何 IOException 包装后抛出。

3.9 未使用的默认配置方法（extractDefaultConfigFile）

  • 源码中存在 extractDefaultConfigFile()：从 /allatori.xml 资源解压到临时目录的 allatori.xml。当前 Mojo 从未调用该方法，配置文件必须通过 configFile 显式指定，无“默认配置”行为。

3.10 异常与失败行为汇总

场景 行为
skip == true 直接 return，打 info 日志，不抛错。
内置 jar 资源不存在 MojoExecutionException("Embedded Allatori jar not found in plugin resources")。
解压/创建临时目录失败 MojoExecutionException("Failed to create temporary directory: ...") 或 "Failed to extract Allatori jar"。
jarFile / configFile 不存在 MojoExecutionException("Allatori jar/file config file not found: ...")。
处理配置文件 IO 异常 MojoExecutionException("Failed to process Allatori config file", e)。
Allatori 子进程退出码非 0 MojoExecutionException("Allatori execution failed with exit code: ...")。
启动子进程或 waitFor 异常 MojoExecutionException("Failed to execute Allatori obfuscator", e)。
任意其他 Exception execute() 的 catch 统一包装为 MojoExecutionException("Error executing Allatori obfuscation", e)。
临时 jar 删除失败 仅 getLog().warn(...)，不抛错。

四、使用说明

4.1 功能特性

  • 支持 Maven 变量替换：${var} 与 @var@（兼容 Spring Boot 资源过滤习惯）。
  • 自动读取项目中的 Allatori 配置文件，并输出“处理后的配置”到构建目录，便于排查。
  • 支持内置 Allatori jar（从插件 resources 解压）或通过 jarFile 指定外置 jar。
  • 配置文件与输出统一使用 UTF-8。
  • 执行完毕后可清理本次解压的临时 jar。

4.2 安装与绑定

在需要使用混淆的 Maven 项目的 pom.xml 中：

<build>
    <plugins>
        <plugin>
            <groupId>com.dv</groupId>
            <artifactId>allatori-maven-plugin</artifactId>
            <version>1.0</version>
            <executions>
                <execution>
                    <id>obfuscate</id>
                    <phase>package</phase>
                    <goals>
                        <goal>obfuscate</goal>
                    </goals>
                </execution>
            </executions>
            <configuration>
                <configFile>${project.basedir}/allatori/allatori.xml</configFile>
            </configuration>
        </plugin>
    </plugins>
</build>
  • 若需跳过混淆（如仅做编译测试）：mvn package -Dallatori.skip=true。

4.3 配置项一览

配置项 说明 默认值 命令行覆盖（-D）
configFile Allatori 配置文件路径（必填） 无，需在 configuration 中指定 -Dallatori.configFile=路径
jarFile Allatori 可执行 jar 路径 不指定则使用插件内置版本（若存在） -Dallatori.jarFile=路径
outputDirectory 混淆输出目录，亦为 Allatori 工作目录 ${project.build.directory} -Dallatori.outputDirectory=路径
skip 是否跳过混淆 false -Dallatori.skip=true
  • 命令行传入的属性会覆盖 pom 中 <configuration> 的同一项，便于本地或 CI 临时切换配置而不改 pom。

4.4 创建 Allatori 配置文件

在项目根目录下创建目录 allatori，并新建 allatori.xml（或其它路径，与 configFile 一致）。示例：

<config>
    <input>
        <jar in="${project.build.finalName}.jar" out="${project.build.finalName}-obfuscated.jar"/>
    </input>
    <keep-names>
        <class access="protected+">
            <field access="protected+"/>
            <method access="protected+"/>
        </class>
        <class template="class *Controller">
            <method template="private+ *(**)" parameters="keep"/>
        </class>
    </keep-names>
    <ignore-classes>
        <class template="class *springframework*"/>
        <class template="class *shardingjdbc*"/>
        <class template="class *jni*"/>
        <class template="class *alibaba*"/>
        <class template="class *persistence*"/>
    </ignore-classes>
</config>
  • in/out 可使用 Maven 变量；Allatori 的工作目录为 outputDirectory，因此 in 一般为 target/<finalName>.jar 对应的文件名（或相对路径）。
  • Spring Boot 项目若使用 @var@ 风格，可写成：in="@project.build.finalName@.jar" out="@project.build.finalName@-obfuscated.jar"。

4.5 内置变量（可由插件替换）

  • project.basedir — 项目根目录绝对路径
  • project.build.directory — 构建输出目录（如 target）
  • project.build.outputDirectory — 编译输出目录（如 target/classes）
  • project.build.finalName — 最终构建产物名（不含扩展名）
  • project.artifactId — 项目 artifactId
  • project.version — 项目版本

以及 pom 中 <properties> 和系统属性中的任意键，只要在配置文件中写成 ${key} 或 @key@ 即可。

  • 变量替换细节：先替换所有 ${...}，再替换所有 @...@；无法解析的占位符保留原样，不会报错。避免在配置中写嵌套变量（如 ${a${b}}），否则可能被截断为 a${b。

4.6 构建与运行

  • 正常打包并混淆：mvn clean package 或 mvn clean install。
  • 插件会在 package 阶段自动执行混淆，控制台可看到处理后的配置文件路径及内容（若开启日志打印）。

4.7 常见问题

现象 处理建议
Allatori config file not found 确认 configFile 指向的路径存在（如 allatori/allatori.xml），或路径中 Maven 变量是否在运行时已解析。
混淆后的 jar 无法运行 检查 keep-names/ignore-classes 等，保留入口类、main 方法及反射/序列化用到的类与成员。
变量未被替换 确认写法为 ${var} 或 @var@，且变量名与 getMavenProperty 支持的一致（或已在 project.properties 中定义）。
内存不足 运行 Maven 时加大 JVM：mvn package -Xms512m -Xmx1024m。Allatori 子进程固定 -Xms128m -Xmx512m，若需调整需修改插件源码 executeAllatori 中的 command 列表。
Embedded Allatori jar not found 插件未打包 lib/allatori.jar 或 classpath 无法加载；请在 allatori-maven-plugin/src/main/resources/lib/ 下放置 allatori.jar，或使用 -Dallatori.jarFile=... 指定外置 jar。
临时 jar 删除失败 多为文件被占用或权限问题；不影响构建结果，仅会打出 warn。可手动清理 java.io.tmpdir/allatori-maven-plugin/。

六、构建与测试插件本身

6.1 技术栈与构建要求

  • JDK：插件源码与编译目标为 Java 8（maven.compiler.source/target: 1.8）。
  • Maven：依赖 maven-plugin-api、maven-plugin-annotations、maven-core（版本 3.8.6），以及 maven-resources-plugin（3.3.1）；打包类型为 maven-plugin，使用 maven-plugin-plugin 3.7.0 生成描述符。
  • Goal 前缀：goalPrefix 为 allatori，即命令行可执行 mvn allatori:obfuscate（若未绑定到生命周期）。
  • 编码：构建与配置文件读写统一 UTF-8（project.build.sourceEncoding 与源码中 "UTF-8"）；子进程标准输出读取使用 JVM 默认编码（未显式传 charset）。

6.2 安装与验证

  • 安装插件到本地仓库（在仓库根目录执行）：
    mvn clean install

  • 仅用测试项目验证：
    cd allatori-plugin-test && mvn clean package
    完成后在 target/ 下查看是否生成混淆后的 jar（由 allatori.xml 的 out 决定）；同时可查看 target/allatori-processed.xml 与控制台日志中的“Processed Allatori config file content”确认变量替换结果。
