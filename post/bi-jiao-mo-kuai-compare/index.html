比较模块（Compare）-技术分享

2938字 2026-03-05
13 min read
Scroll Down

1. 模块介绍

比较模块提供了强大的对象差异对比能力，能够智能识别并展示两个对象之间的属性差异。该模块通过注解驱动的方式，支持自定义比较规则、字段映射和差异格式化，适用于数据审计、变更日志记录、版本对比等场景。

2. 核心优势（为什么选择本模块）

优势 说明
零侵入业务逻辑 仅通过注解标记参与比较的字段，无需在业务代码中手写逐字段比较，避免重复、易错的 if-else 判断。
可读可维护的差异结果 输出统一的「修改前 → 修改后」格式，并支持字段中文描述与枚举/映射后的可读值，便于直接写入操作日志或展示给用户。
灵活的映射与忽略 支持字符串规则、枚举规则、@CompareMapping 注解三种映射方式；支持按字段忽略、按类忽略，并默认忽略常见审计字段（createTime/createUser/modifyTime/modifyUser）。
列表与集合比较 除单对象比较外，支持按索引顺序比较列表、按映射键（如 ID）比较集合，满足「列表前后快照」类审计需求。
国际化友好 与 Spring 消息源集成，字段描述支持 i18n key，可根据请求语言输出多语言比较结果。
性能友好 类级别 CompareCore 实例池化、字段信息缓存，同一类型重复比较时避免重复反射与实例创建。
与 Spring Boot 无缝集成 提供自动配置与 CompareProperties，引入依赖即可使用，支持通过配置文件定制主语言与资源前缀。

典型适用场景：操作审计日志、数据变更记录、配置/版本 diff 展示、后台「修改前/修改后」对比等。若你当前通过手写字段比较或拼字符串来记录变更，本模块可显著减少样板代码并统一输出格式。

与手写比较对比：手写需对每个字段判空、equals、拼字符串，易漏字段、格式不统一、枚举码值需自行转文案；本模块通过注解声明参与比较的字段与映射规则，一次配置、多处复用，输出格式统一且支持 i18n，更适合中大型项目中的审计与日志需求。

3. 核心功能

  • 对象属性差异对比：精确比较两个对象的属性差异，并格式化为可读的「修改前 → 修改后」描述。
  • 列表与集合比较：支持按索引顺序比较列表、按映射键（如 ID）比较集合，适配有序/无序两种场景。
  • 多种字段值映射：支持字符串规则（ruleStr）、枚举规则（ruleEnum）、注解映射（@CompareMapping），将码值转为业务可读文案。
  • 灵活的忽略机制：支持类/字段级 @CompareIgnore、方法级忽略列表，并默认忽略常见审计字段。
  • 国际化：字段描述支持 i18n key，结合请求语言输出多语言比较结果。
  • 性能优化：CompareCore 实例池化、字段信息缓存，减少重复反射与对象创建。

4. 核心类与接口

4.1 工具类

CompareUtil

主要的工具类，提供了获取比较实例和核心比较功能。采用单例模式设计，内部使用缓存池存储比较实例。

// 获取指定类的比较实例
CompareCore compareCore = CompareUtil.forClazz(User.class);

CompareCore

由 CompareUtil.forClazz(Class) 返回的核心比较实现类，提供以下方法：

  • compare(T source, T target): 比较两个对象的差异
  • compareList(List<T> sourceList, List<T> targetList): 按索引顺序比较列表
  • compareArr(Collection<T> sourceList, Collection<T> targetList, Function<T, ?> mappingRule): 基于映射规则比较集合

CompareNode

用于存储比较过程中的字段信息，包含字段名、字段值和字段描述。

public class CompareNode {
    private String fieldKey;    // 字段名
    private Object fieldValue;  // 字段值
    private String fieldDesc;   // 字段描述
}

4.2 注解

@CompareProperty

标记需要进行比较的字段，支持配置字段描述和多种映射规则。

@Target({ElementType.METHOD, ElementType.FIELD})
@Retention(RetentionPolicy.RUNTIME)
public @interface CompareProperty {
    String value() default "";  // 字段描述（可填 i18n key）
    CompareRule ruleType() default CompareRule.STR;  // 映射规则类型
    String[] ruleStr() default "";  // 字符串映射规则（STR）
    Class<? extends Enum<? extends Function>> ruleEnum() default DefaultRuleEnum.class;  // 枚举映射规则（ENUM）
    CompareMapping[] ruleMappings() default {};  // 注解映射规则（MAPPING）
}

@CompareIgnore

标记需要忽略比较的类或字段，被标记的将不参与差异计算。

@Retention(RetentionPolicy.RUNTIME)
@Target({ElementType.TYPE, ElementType.METHOD})
public @interface CompareIgnore {
}

@CompareMapping

与 @CompareProperty(ruleType = CompareRule.MAPPING, ruleMappings = {...}) 配合使用，声明「源值 → 目标展示值」的映射，无需单独写枚举类。

public @interface CompareMapping {
    String source();  // 源值（如 "0"）
    String target();  // 展示值（如 "禁用"）
}

4.3 枚举

CompareRule

定义字段映射规则类型：

public enum CompareRule {
    STR,     // 字符串规则，配合 ruleStr
    ENUM,    // 枚举规则，配合 ruleEnum
    MAPPING  // 注解映射规则，配合 ruleMappings
}

5. 快速开始

5.1 依赖与集成

在 Spring Boot 项目中引入本模块依赖后，自动配置会生效；可通过 application.yml 或 application.properties 可选配置：

spring:
  ai:
    framework:
      compare:
        primary-language: zh                    # 主语言，未从请求头解析时使用
        message-source-basename: compare       # 国际化资源文件前缀

5.2 基础配置

在需要比较的类的字段上添加 @CompareProperty 注解：

@Data
public class User {
    @CompareProperty("用户ID")
    private Long id;
    
    @CompareProperty("用户名")
    private String username;
    
    @CompareProperty("状态")
    private Integer status;
    
    // 不需要比较的字段不加注解
    private Date createTime;
}

5.3 对象比较示例

// 创建对象
User oldUser = new User(1L, "张三", 1);
User newUser = new User(1L, "李四", 0);

// 获取比较实例并执行比较
CompareCore compareCore = CompareUtil.forClazz(User.class);
String diffResult = compareCore.compare(oldUser, newUser);

// 输出结果：
// [用户名] 修改前:张三 ——> 修改后:李四
// [状态] 修改前:1 ——> 修改后:0
System.out.println(diffResult);

6. 高级功能

6.1 字段值映射

6.1.1 字符串映射规则

通过ruleStr参数配置简单的键值对映射，规则基于字符->进行键值分隔：

@CompareProperty(
    value = "状态",
    ruleStr = {
        "0->禁用",
        "1->启用"
    }
)
private Integer status;
  • 格式说明：原始值->映射后的值，其中->是固定的分隔符
  • 比较模块会自动解析这些规则，并在比较结果中显示映射后的值

比较结果将显示映射后的值：

[状态] 修改前:启用 ——> 修改后:禁用

6.1.2 ruleType 属性使用说明

ruleType 用于指定字段值的映射规则类型，支持三种规则：

  • CompareRule.STR：字符串映射规则（默认），与 ruleStr 配合使用。
  • CompareRule.ENUM：枚举映射规则，与 ruleEnum 配合使用。
  • CompareRule.MAPPING：注解映射规则，与 ruleMappings（@CompareMapping）配合使用，无需单独写枚举类。

使用规则说明：

  1. 不指定 ruleType 时，默认为 CompareRule.STR。
  2. 配置了 ruleStr 时，会按字符串规则处理（可省略显式写 ruleType=STR）。
  3. 使用枚举或注解映射时，建议显式设置 ruleType=ENUM 或 ruleType=MAPPING 以提升可读性。
  4. 优先级：若同时配置多种规则，按实现约定以 STR > MAPPING > ENUM 等顺序生效，建议单字段只选一种方式。

示例：

// 明确指定使用字符串映射规则
@CompareProperty(
    value = "状态",
    ruleType = CompareRule.STR,
    ruleStr = {
        "0->禁用",
        "1->启用"
    }
)
private Integer status;

// 明确指定使用枚举映射规则
@CompareProperty(
    value = "状态",
    ruleType = CompareRule.ENUM,
    ruleEnum = StatusEnum.class
)
private Integer status;

6.1.3 枚举映射规则

通过实现Function接口的枚举类实现更复杂的映射：

  1. 定义映射枚举类：
@Getter
@AllArgsConstructor
public enum StatusEnum implements Function<Object, String> {
    DISABLED(0, "禁用"),
    ENABLED(1, "启用");
    
    private Integer code;
    private String desc;
    
    public static StatusEnum getByCode(Integer code) {
        for (StatusEnum value : StatusEnum.values()) {
            if (value.code.equals(code)) {
                return value;
            }
        }
        return null;
    }
    
    @Override
    public String apply(Object o) {
        return getByCode((Integer) o).getDesc();
    }
}
  1. 在注解中使用枚举映射：
@CompareProperty(
    value = "状态",
    ruleType = CompareRule.ENUM,
    ruleEnum = StatusEnum.class
)
private Integer status;

6.1.4 注解映射规则（@CompareMapping）

不需要单独建枚举类时，可直接在注解上写「源值 → 展示值」：

@CompareProperty(
    value = "状态",
    ruleType = CompareRule.MAPPING,
    ruleMappings = {
        @CompareMapping(source = "0", target = "禁用"),
        @CompareMapping(source = "1", target = "启用")
    }
)
private Integer status;

比较结果会显示「禁用」「启用」等目标文案，便于日志与前端展示。

6.2 列表比较

6.2.1 按索引比较列表

适用于顺序敏感的列表比较：

List<User> oldList = Arrays.asList(new User(1L, "张三", 1));
List<User> newList = Arrays.asList(new User(1L, "李四", 0));

String diffResult = CompareUtil.forClazz(User.class).compareList(oldList, newList);

6.2.2 基于映射规则比较集合

适用于无序集合比较，通过映射规则确定元素对应关系：

List<User> oldList = Arrays.asList(new User(1L, "张三", 1));
List<User> newList = Arrays.asList(new User(1L, "李四", 0));

// 以id作为映射规则进行比较
String diffResult = CompareUtil.forClazz(User.class).compareArr(
    oldList, newList, User::getId
);

6.3 忽略特定字段

比较时可以指定忽略某些字段：

List<String> ignoreFields = Arrays.asList("status", "createTime");
String diffResult = CompareUtil.forClazz(User.class).compare(
    oldUser, newUser, ignoreFields
);

注意：模块默认会忽略以下审计字段，无需手动配置：

  • createTime
  • createUser
  • modifyTime
  • modifyUser

7. 国际化（可选）

  • 在 @CompareProperty 的 value 中填写 i18n 的 key（如 user.status），模块会通过 CompareI18n 从配置的 MessageSource 中取当前语言文案。
  • 请求头 Accept-Language 或配置项 spring.ai.framework.compare.primary-language 决定语言；资源文件前缀由 spring.ai.framework.compare.message-source-basename 指定（默认 compare），如 compare_zh.properties、compare_en.properties。
  • 未配置 i18n 时，可直接在 value 中写中文描述，不影响使用。

8. 性能优化

该模块通过以下机制优化性能：

  1. 实例缓存：使用INSTANCE_POOL缓存CompareCore实例
  2. 字段缓存：使用CLAZZ_FIELD_CACHE缓存类的字段信息
  3. 懒加载：仅在需要时才解析字段信息

9. 使用注意事项

  1. 类与字段要求：

    • 被比较的类需要有默认构造函数（用于创建空对象）
    • 需要比较的字段必须添加@CompareProperty注解
    • 嵌套对象暂不支持递归比较，只支持JDK基础类型

  2. 异常处理：

    • 当比较的两个对象类型不匹配时，会抛出IllegalArgumentException
    • 枚举映射规则配置不正确时可能抛出异常

  3. 性能考虑：

    • 对于大型对象或复杂集合，建议只比较必要的字段
    • 频繁比较相同类型对象时，实例缓存会自动提高性能

10. 完整示例

下面是一个完整的使用示例：

// 1. 定义比较规则枚举
@Getter
@AllArgsConstructor
public enum StatusEnum implements Function<Object, String> {
    DISABLED(0, "禁用"),
    ENABLED(1, "启用");
    
    private Integer code;
    private String desc;
    
    public static StatusEnum getByCode(Integer code) {
        for (StatusEnum value : StatusEnum.values()) {
            if (value.code.equals(code)) {
                return value;
            }
        }
        return null;
    }
    
    @Override
    public String apply(Object o) {
        return getByCode((Integer) o).getDesc();
    }
}

// 2. 定义需要比较的类
@Data
public class User {
    @CompareProperty("用户ID")
    private Long id;
    
    @CompareProperty("用户名")
    private String username;
    
    @CompareProperty(
        value = "状态",
        ruleType = CompareRule.ENUM,
        ruleEnum = StatusEnum.class
    )
    private Integer status;
    
    @CompareProperty(
        value = "类型",
        ruleStr = {
            "0->普通用户",
            "1->管理员"
        }
    )
    private Integer type;
    
    // 不参与比较的字段
    private Date createTime;
}

// 3. 执行比较
public class CompareDemo {
    public static void main(String[] args) {
        // 创建对象
        User oldUser = new User(1L, "张三", 1, 0);
        User newUser = new User(1L, "李四", 0, 1);
        
        // 执行比较
        String diffResult = CompareUtil.forClazz(User.class).compare(oldUser, newUser);
        
        // 输出结果
        System.out.println("对象差异比较结果：");
        System.out.println(diffResult);
    }
}

输出结果：

对象差异比较结果：
[用户名] 修改前:张三 ——> 修改后:李四
[状态] 修改前:启用 ——> 修改后:禁用
[类型] 修改前:普通用户 ——> 修改后:管理员
